description: "Reference for the Effects interface — the capability system bridging service TypeScript to the StartOS runtime for networking, storage, health, and SSL."
---

The `Effects` interface is the sole channel through which service code reaches the StartOS runtime. Every lifecycle hook — `main`, `init`, `uninit`, `createBackup`, and actions — receives an `Effects` object as its entry point. Rather than making direct system calls, your service code calls methods on `Effects`, which the runtime mediates, enforces, and records. This design makes the runtime boundary explicit, enables the OS to supervise all side effects, and lets you substitute a mock `Effects` in tests.

The interface is defined in `base/lib/Effects.ts` and is organized into ten subsystems.

<Note>
  Most `Effects` methods are accessed indirectly through the typed wrappers on the `StartSdk` facade (e.g. `sdk.restart`, `sdk.setHealth`). Use the raw `Effects` object directly only when you need a method not yet surfaced by the facade.
</Note>

## Reactive subscription pattern

Many `Effects` methods accept an optional `callback` parameter. When provided, the runtime calls your callback whenever the underlying value changes. The SDK wraps this low-level mechanism in the `Watchable` base class, which exposes four higher-level strategies:

| Method | Behavior |
|--------|----------|
| `const()` | Read once. If the value changes, triggers a retry of the enclosing reactive context. |
| `once()` | Read once with no reactivity. |
| `watch(abort?)` | Returns an async generator that yields on each change. |
| `onChange(callback)` | Invokes a callback on each change; return `{ cancel: true }` to stop. |
| `waitFor(predicate)` | Blocks until the predicate returns `true` for a yielded value. |

The following reactive wrappers are pre-built around common `Effects` methods: `GetContainerIp`, `GetStatus`, `GetSystemSmtp`, `GetOutboundGateway`, `GetSslCertificate`, `GetHostInfo`, `GetServiceManifest`. The `StartSdk` facade exposes these as `sdk.getContainerIp(effects)`, `sdk.getStatus(effects)`, etc., returning a `Watchable` instance.

```typescript
// Read once, retry context on change
const ip = await sdk.getContainerIp(effects).const()

// Stream all future changes
for await (const ip of sdk.getContainerIp(effects).watch()) {
  console.log('Container IP changed:', ip)
}

// Callback on change, cancellable
sdk.getContainerIp(effects).onChange((ip) => {
  configFile.merge({ proxyTarget: ip })
  return { cancel: false }
})

// Block until non-loopback IP
const ip = await sdk.getContainerIp(effects).waitFor(
  (ip) => !ip.startsWith('127.')
)
```

## Action subsystem

Registered actions appear in the StartOS UI where users can trigger them. All action registration should happen during `init`.

<AccordionGroup>
  <Accordion title="effects.action.export">
    Register an action with the OS so it appears in the user interface.

    ```typescript
    effects.action.export(options: {
      id: ActionId
      metadata: ActionMetadata
    }): Promise<null>
    ```

    <ParamField path="options.id" type="ActionId" required>
      Unique string identifier for the action within this service.
    </ParamField>
    <ParamField path="options.metadata" type="ActionMetadata" required>
      Name, description, visibility, allowed statuses, warning, group, and `hasInput` flag.
    </ParamField>
  </Accordion>

  <Accordion title="effects.action.clear">
    Remove all registered actions except those in the `except` list. Call after exporting all actions during `init` to remove any stale entries from a previous version.

    ```typescript
    effects.action.clear(options: {
      except: ActionId[]
    }): Promise<null>
    ```

    <ParamField path="options.except" type="ActionId[]" required>
      Action IDs to keep. All others are removed.
    </ParamField>
  </Accordion>

  <Accordion title="effects.action.getInput">
    Retrieve the current input for an action, optionally from another package.

    ```typescript
    effects.action.getInput(options: {
      packageId?: PackageId
      actionId: ActionId
    }): Promise<ActionInput | null>
    ```

    <ParamField path="options.packageId" type="PackageId">
      Target package. Defaults to the current service.
    </ParamField>
    <ParamField path="options.actionId" type="ActionId" required>
      The action whose input to retrieve.
    </ParamField>
  </Accordion>

  <Accordion title="effects.action.run">
    Programmatically execute an action, optionally on another package.

    ```typescript
    effects.action.run<Input extends Record<string, unknown>>(options: {
      packageId?: PackageId
      actionId: ActionId
      input?: Input
    }): Promise<ActionResult | null>
    ```

    <ParamField path="options.packageId" type="PackageId">
      Target package. Defaults to the current service.
    </ParamField>
    <ParamField path="options.actionId" type="ActionId" required>
      The action to execute.
    </ParamField>
    <ParamField path="options.input" type="Input">
      Optional input values to pass to the action.
    </ParamField>
  </Accordion>

  <Accordion title="effects.action.createTask">
    Create a task notification that appears in the StartOS task center, prompting the user to run an action.

    ```typescript
    effects.action.createTask(options: CreateTaskParams): Promise<null>
    ```

    <ParamField path="options" type="CreateTaskParams" required>
      Includes `packageId`, `actionId`, `severity`, `replayId`, and optional pre-fill data. See `osBindings` for the full `CreateTaskParams` type.
    </ParamField>
  </Accordion>

  <Accordion title="effects.action.clearTasks">
    Remove task notifications by replay ID.

    ```typescript
    effects.action.clearTasks(
      options: { only: string[] } | { except: string[] }
    ): Promise<null>
    ```

    <ParamField path="options" type="object" required>
      Pass `{ only: [...replayIds] }` to remove specific tasks, or `{ except: [...replayIds] }` to remove all others.
    </ParamField>
  </Accordion>
</AccordionGroup>

## Control subsystem

Service lifecycle control — restart, shutdown, and status queries.

<AccordionGroup>
  <Accordion title="effects.restart">
    Restart this service's main function.

    ```typescript
    effects.restart(): Promise<null>
    ```
  </Accordion>

  <Accordion title="effects.shutdown">
    Stop this service's main function.

    ```typescript
    effects.shutdown(): Promise<null>
    ```
  </Accordion>

  <Accordion title="effects.getStatus">
    Query the current runtime status of a service. Accepts an optional `callback` for reactive subscriptions.

    ```typescript
    effects.getStatus(options: {
      packageId?: PackageId
      callback?: () => void
    }): Promise<StatusInfo | null>
    ```

    <ParamField path="options.packageId" type="PackageId">
      Target package. Defaults to the current service.
    </ParamField>
    <ParamField path="options.callback" type="() => void">
      Called when the status changes. Use `sdk.getStatus(effects)` for the full reactive API.
    </ParamField>

    <Tip>
      Prefer `sdk.getStatus(effects).const()` over calling this directly.
    </Tip>
  </Accordion>

  <Accordion title="effects.setMainStatus">
    Indicate to the OS what run-state the service is in.

    ```typescript
    effects.setMainStatus(options: SetMainStatus): Promise<null>
    ```

    <ParamField path="options" type="SetMainStatus" required>
      The run-state to report. See `osBindings/SetMainStatus` for the valid values.
    </ParamField>

    <Warning>
      This method is deprecated. The SDK daemon management system sets the main status automatically.
    </Warning>
  </Accordion>
</AccordionGroup>

## Plugin subsystem

The plugin subsystem is gated by the `plugins` field in your service manifest. Methods are only accessible when the relevant plugin is declared. The `url-v0` plugin enables URL management — exporting clickable addresses to the StartOS UI.

## SSL subsystem

Obtain TLS certificates and private keys issued by the StartOS certificate authority.

<AccordionGroup>
  <Accordion title="effects.getSslCertificate">
    Return a PEM-encoded full certificate chain for the given hostnames. Accepts an optional `callback` for reactive subscriptions.

    ```typescript
    effects.getSslCertificate(options: {
      hostnames: string[]
      algorithm?: 'ecdsa' | 'ed25519'
      callback?: () => void
    }): Promise<[string, string, string]>
    ```

    <ParamField path="options.hostnames" type="string[]" required>
      List of hostnames (e.g. `.onion` addresses or LAN hostnames) to include in the certificate.
    </ParamField>
    <ParamField path="options.algorithm" type="'ecdsa' | 'ed25519'">
      Key algorithm preference. Defaults to ECDSA.
    </ParamField>
    <ParamField path="options.callback" type="() => void">
      Called when the certificate is renewed. Use `sdk.getSslCertificate(effects, hostnames)` for the full reactive API.
    </ParamField>

    Returns a tuple of `[certificate, intermediates, root]` in PEM format.
  </Accordion>

  <Accordion title="effects.getSslKey">
    Return the PEM-encoded private key corresponding to the SSL certificate for the given hostnames.

    ```typescript
    effects.getSslKey(options: {
      hostnames: string[]
      algorithm?: 'ecdsa' | 'ed25519'
    }): Promise<string>
    ```

    <ParamField path="options.hostnames" type="string[]" required>
      Hostnames that the key covers. Must match a prior `getSslCertificate` call.
    </ParamField>
    <ParamField path="options.algorithm" type="'ecdsa' | 'ed25519'">
      Key algorithm. Must match the algorithm used in `getSslCertificate`.
    </ParamField>
  </Accordion>
</AccordionGroup>

## System subsystem

System-wide configuration access and data version management.

<AccordionGroup>
  <Accordion title="effects.getSystemSmtp">
    Return the globally configured SMTP settings, if the user has provided them. Accepts an optional `callback` for reactive subscriptions.

    ```typescript
    effects.getSystemSmtp(options: {
      callback?: () => void
    }): Promise<SmtpValue | null>
    ```

    Returns `null` if no system SMTP is configured. The `SmtpValue` type contains `host`, `port`, `from`, `username`, `password`, and `security`.

    <Tip>
      Prefer `sdk.getSystemSmtp(effects).const()` to automatically retry your config generation when SMTP settings change.
    </Tip>
  </Accordion>

  <Accordion title="effects.setDataVersion">
    Persist the data migration version for this service. Call at the end of each migration step during `init`.

    ```typescript
    effects.setDataVersion(options: {
      version: string | null
    }): Promise<null>
    ```

    <ParamField path="options.version" type="string | null" required>
      An ExVer version string (e.g. `'1.2.0:0'`) identifying how far data has been migrated, or `null` to clear it.
    </ParamField>
  </Accordion>

  <Accordion title="effects.getDataVersion">
    Return the data migration version previously set by `setDataVersion`.

    ```typescript
    effects.getDataVersion(): Promise<string | null>
    ```

    Returns `null` if no version has been set (i.e. fresh install with no prior migration).
  </Accordion>
</AccordionGroup>
